# LazyVim 开发贡献指南

*为开发者和贡献者提供的全面开发文档*

---

## 概述

LazyVim 是一个开源的 Neovim 配置框架，欢迎社区贡献。本指南涵盖代码规范、开发环境设置、插件开发、测试和贡献流程。

**贡献类型**：
- 🐛 **Bug修复** - 修复已知问题
- ✨ **新功能** - 添加Core插件或工具函数
- 📦 **Extra插件** - 添加可选功能模块
- 📚 **文档改进** - 完善使用文档和代码注释
- 🔧 **性能优化** - 提升启动速度和运行效率

## 开发环境设置

### 环境要求

**必需工具**：
- **Neovim >= 0.11.0** (LuaJIT构建)
- **Git >= 2.19.0**
- **stylua** - Lua代码格式化工具
- **selene** - Lua静态分析工具

**推荐工具**：
- **luarocks** - Lua包管理器
- **ripgrep (rg)** - 快速文本搜索
- **fd** - 快速文件查找

### 开发环境初始化

```bash
# 1. 克隆仓库
git clone https://github.com/LazyVim/LazyVim.git
cd LazyVim

# 2. 安装代码质量工具
# 使用包管理器安装stylua和selene
# macOS with Homebrew:
brew install stylua selene

# 或使用Cargo:
cargo install stylua selene

# 3. 设置开发用的Neovim配置
export NVIM_APPNAME=lazyvim-dev
mkdir -p ~/.config/lazyvim-dev
# 创建开发配置链接到项目目录
```

### 项目结构理解

```
LazyVim/
├── lua/lazyvim/
│   ├── config/          # 核心配置模块
│   ├── plugins/         # 插件规范和extras
│   └── util/           # 工具函数库
├── tests/              # 测试套件
├── scripts/            # 构建和测试脚本
├── .github/workflows/  # CI/CD配置
├── stylua.toml        # 代码格式化配置
├── selene.toml        # 静态分析配置
└── doc/              # 帮助文档
```

## 代码规范

### Lua代码风格 (stylua.toml)

```toml
# 基于项目根目录的stylua.toml配置
column_width = 120          # 最大行宽
line_endings = "Unix"       # 使用Unix换行符  
indent_type = "Spaces"      # 使用空格缩进
indent_width = 2           # 2空格缩进
quote_style = "AutoPreferDouble"  # 优先使用双引号

[sort_requires]
enabled = true             # 自动排序require语句
```

**格式化命令**：
```bash
# 格式化所有Lua文件
stylua .

# 检查格式化（不修改文件）  
stylua --check .

# 格式化特定文件
stylua lua/lazyvim/plugins/example.lua
```

### 静态分析 (selene.toml)

```toml
# 基于项目的selene.toml配置
std = "vim"               # 使用Vim标准库
mixed_table = "allow"     # 允许混合表结构
```

**分析命令**：
```bash
# 分析所有Lua文件
selene .

# 分析特定目录
selene lua/lazyvim/plugins/

# 生成详细报告
selene . --display-style=json
```

### 命名约定

**模块命名**：
```lua
-- ✅ 正确：使用点分隔的小写名称
local M = {}

-- ✅ 正确：描述性的局部变量名
local lazy_util = require("lazy.util")
local extra_manager = require("lazyvim.util.extras")
```

**函数命名**：
```lua
-- ✅ 正确：snake_case函数名
function M.get_plugin(name)
function M.has_extra(extra)
function M.setup_keymaps()

-- ❌ 错误：camelCase或PascalCase
function M.getPlugin()
function M.SetupKeymaps()
```

**常量命名**：
```lua
-- ✅ 正确：大写蛇形命名
local DEFAULTS = {}
local MAX_RETRIES = 3

-- ❌ 错误：小写或混合大小写
local defaults = {}  -- 这是变量，不是常量
local maxRetries = 3
```

### 类型注解

LazyVim 使用详细的 LuaLS 类型注解：

```lua
---@class LazyPlugin: LazyPluginBase
---@field dependencies? LazyPlugin[]

---@class LazyVimConfig
---@field extras string[]

---@param plugin string|LazyPlugin
---@param opts table?
---@return boolean
function M.has(plugin, opts)
  return M.get_plugin(plugin) ~= nil
end
```

## 插件开发规范

### Core插件 vs Extra插件区别

| 特性 | Core插件 | Extra插件 |
|------|----------|-----------|
| **加载方式** | 自动加载 | 用户选择启用 |
| **用途** | 基础功能 | 可选功能增强 |
| **配置要求** | 必须可覆盖 | 完整独立配置 |
| **质量标准** | 极高稳定性 | 高质量但可实验性 |
| **文件位置** | `plugins/*.lua` | `plugins/extras/**/*.lua` |

### Core插件开发规范

**基本结构**：
```lua
-- lua/lazyvim/plugins/coding.lua
return {
  -- 核心插件规范
  {
    "plugin-name/plugin-repo",
    event = "VeryLazy",           -- 适当的懒加载策略
    opts = {},                    -- 默认配置选项
    config = function(_, opts)    -- 配置函数
      require("plugin").setup(opts)
    end,
  },
  
  -- 条件插件（基于用户配置）
  vim.g.some_feature_enabled and {
    "conditional/plugin",
    opts = {},
  } or nil,
}
```

**Core插件要求**：
```lua
-- ✅ 必需：避免Vim插件，优先Lua插件
-- ✅ 必需：所有配置必须可被用户覆盖
-- ✅ 必需：实现适当的懒加载策略
-- ✅ 必需：Optional插件须标记 optional = true

return {
  {
    "hrsh7th/nvim-cmp",
    optional = true,              -- 仅在用户安装时启用
    opts = function(_, opts)
      -- 修改现有配置而不替换
      table.insert(opts.sources, 1, { name = "copilot" })
    end,
  },
}
```

### Extra插件开发规范

**必需结构**：
```lua
-- lua/lazyvim/plugins/extras/category/plugin.lua
return {
  -- ✅ 必需字段：推荐检查
  recommended = function()
    return LazyVim.extras.wants({
      ft = { "python", "typescript" },          -- 文件类型检测
      root = { "package.json", "pyproject.toml" } -- 项目根文件检测
    })
  end,
  
  -- TreeSitter语言支持（Lang extras必需）
  {
    "nvim-treesitter/nvim-treesitter",
    opts = { ensure_installed = { "python", "typescript" } },
  },
  
  -- LSP服务器配置（Lang extras推荐）
  {
    "neovim/nvim-lspconfig",
    opts = {
      servers = {
        pyright = {
          settings = {
            python = {
              analysis = { typeCheckingMode = "basic" },
            },
          },
        },
      },
    },
  },
  
  -- 主要插件规范
  {
    "main-plugin/repo",
    optional = true,        -- 仅在用户安装时启用
    opts = {},             -- 可被用户覆盖的配置
    config = function(_, opts)
      -- 完整的配置逻辑
    end,
  },
}
```

**Extra插件质量要求**：

1. **必须是知名且需要大量配置的插件**
2. **简单的单行配置不被接受**
3. **包含完整的依赖关系管理**
4. **Lang extra必须有recommended字段**
5. **不能重复设置已存在的图标**
6. **不能安装默认的TreeSitter语言**

### 依赖管理最佳实践

```lua
-- ✅ 正确：Lua依赖作为单独规范添加
return {
  -- Lua库依赖
  { "lua-library/repo", lazy = true },
  
  -- 主插件
  {
    "main-plugin/repo",
    dependencies = {
      "binary-tool/repo",  -- 二进制工具依赖
    },
    -- 不要在dependencies中指定Lua依赖
  },
}
```

## 测试

### 测试框架

LazyVim使用自定义的测试框架，基于Neovim的minitest：

**运行测试**：
```bash
# 运行完整测试套件
./scripts/test

# 直接运行测试（等价命令）
nvim -l tests/minit.lua --minitest

# 运行特定测试文件
nvim -l tests/minit.lua tests/extras/extra_spec.lua
```

### 测试结构

```
tests/
├── minit.lua            # 测试入口和环境初始化
├── extras/extra_spec.lua # Extra插件测试
├── util/util_spec.lua   # 工具函数测试
└── dd_spec.lua         # 其他功能测试
```

### 编写测试

**Extra插件测试模式**：
```lua
-- tests/extras/my_extra_spec.lua
describe("My Extra", function()
  it("spec is valid", function()
    local spec = require("lazyvim.plugins.extras.category.my-extra")
    assert.is_table(spec)
    -- 验证插件规范结构
  end)
  
  it("has recommended set for lang extra", function()
    local spec = require("lazyvim.plugins.extras.lang.my-language")
    assert.is_not_nil(spec.recommended)
    -- Lang extra必须有recommended字段
  end)
  
  it("does not install default treesitter langs", function()
    -- 确保不重复安装默认语言
  end)
  
  it("does not set existing icons", function()
    -- 确保不覆盖现有图标
  end)
end)
```

**工具函数测试模式**：
```lua  
-- tests/util/my_util_spec.lua
describe("My Util", function()
  local my_util = require("lazyvim.util.my-util")
  
  it("exports expected functions", function()
    assert.is_function(my_util.setup)
    assert.is_function(my_util.get)
  end)
  
  it("handles edge cases", function()
    assert.is_nil(my_util.get("nonexistent"))
    assert.has_no.errors(function()
      my_util.setup({})
    end)
  end)
end)
```

### 测试覆盖要求

所有新功能必须包含测试：

1. **插件规范有效性测试**
2. **配置选项验证测试** 
3. **错误处理测试**
4. **边界条件测试**
5. **向后兼容性测试**

## CI/CD流程

### GitHub Actions配置

LazyVim使用共享工作流配置（`.github/workflows/ci.yml`）：

```yaml
name: CI
on:
  push:
    branches: [main, master]
  pull_request:

jobs:
  ci:
    uses: folke/github/.github/workflows/ci.yml@main
    secrets: inherit
    with:
      plugin: LazyVim
      repo: LazyVim/LazyVim
```

### CI检查项目

自动化检查包括：

1. **代码格式化检查** - `stylua --check .`
2. **静态分析** - `selene .`
3. **测试执行** - `./scripts/test`
4. **类型检查** - LuaLS类型验证
5. **插件规范验证** - Extra插件结构检查
6. **文档生成测试** - `nvim --headless -c "helptags doc" -c quit`

### Pull Request要求

提交PR前必须确保：

```bash
# 1. 格式化代码
stylua .

# 2. 静态分析通过
selene .

# 3. 运行测试套件
./scripts/test

# 4. 生成文档标签
nvim --headless -c "helptags doc" -c quit

# 5. 检查Git状态
git status  # 确保没有未提交的格式化更改
```

```bash
git checkout -b feature/my-awesome-feature
```

### 2. 开发阶段

**代码开发**：
```bash
# 1. 进行开发工作
# 编辑文件，添加功能，修复bug

# 2. 运行质量检查
stylua .
selene . 
./scripts/test

# 3. 提交更改
git add .

```

### 3. 提交阶段

**Pull Request**：
```bash
# 1. 推送到你的Fork
git push origin feature/my-awesome-feature

# 2. 在GitHub上创建Pull Request
# 3. 填写PR模板，描述更改内容
# 4. 等待代码审查和CI检查
```

### PR模板内容

```markdown

- [ ] Bug修复
- [ ] 新功能  
- [ ] Extra插件
- [ ] 文档更新
- [ ] 性能优化

## 变更描述
简要描述你的更改内容和原因

## 测试
- [ ] 已添加相应测试
- [ ] 所有测试通过
- [ ] 手动测试验证

## 检查清单
- [ ] 代码已格式化（stylua）
- [ ] 静态分析通过（selene）
- [ ] 文档已更新
- [ ] 向后兼容性考虑
```

### 4. 代码审查

**审查关键点**：

1. **架构合理性** - 是否遵循LazyVim设计模式
2. **代码质量** - 可读性、可维护性
3. **性能影响** - 是否影响启动速度
4. **向后兼容** - 是否破坏现有配置
5. **测试覆盖** - 是否有充分的测试
6. **文档完整** - 是否需要更新文档

## 专项开发指南

### 添加新的Extra插件

**1. 选择合适的分类**：
```bash
lua/lazyvim/plugins/extras/
├── ai/           # AI工具
├── coding/       # 编程辅助
├── editor/       # 编辑器增强  
├── formatting/   # 代码格式化
├── lang/         # 语言支持
├── linting/      # 代码检查
├── lsp/          # LSP增强
├── test/         # 测试工具
├── ui/           # 界面主题
└── util/         # 实用工具
```

**2. 创建插件文件**：
```lua
-- lua/lazyvim/plugins/extras/category/my-plugin.lua
return {
  recommended = function()
    -- 智能推荐逻辑
    return LazyVim.extras.wants({
      ft = { "specific-filetype" },
      root = { "project-marker-file" },
    })
  end,
  
  {
    "author/plugin-name",
    optional = true,
    opts = {
      -- 默认配置
    },
    keys = {
      -- 键位映射
    },
    config = function(_, opts)
      -- 完整配置逻辑
    end,
  },
}
```

**3. 添加测试**：
```lua
-- tests/extras/my_plugin_spec.lua
describe("My Plugin Extra", function()
  it("spec is valid", function()
    local spec = require("lazyvim.plugins.extras.category.my-plugin")
    assert.is_table(spec)
  end)
end)
```

### 添加工具函数

**1. 选择模块**：
```lua
-- lua/lazyvim/util/my-module.lua
local M = {}

---@param option string
---@return boolean
function M.check(option)
  -- 实现逻辑
  return true
end

function M.setup()
  -- 初始化逻辑  
end

return M
```

**2. 更新索引文件**（如需要）：
```lua
-- lua/lazyvim/util/init.lua中会自动通过元表加载
-- 无需手动添加，M.my-module 会自动可用
```

### 修改核心配置

**谨慎修改核心配置**，确保向后兼容：

```lua  
-- lua/lazyvim/config/options.lua
local options = {
  -- 添加新的默认选项
  new_option = true,
}

-- 确保用户可以覆盖
for k, v in pairs(options) do
  if vim.opt[k] ~= nil then
    vim.opt[k] = v
  end
end
```

## 高级主题

### 错误处理模式

```lua
-- 使用LazyVim.try进行错误恢复
LazyVim.try(function()
  require("some-plugin").setup(opts)
end, {
  msg = "Failed to setup plugin",
  on_error = function(msg)
    LazyVim.warn("Plugin setup failed: " .. msg)
  end,
})
```

### 性能优化技巧

```lua
-- 1. 延迟加载昂贵操作
local expensive_operation = LazyVim.lazy(function()
  return require("heavy-module").compute()
end)

-- 2. 使用适当的加载事件
{
  "plugin/name",
  event = "VeryLazy",     -- 延迟到启动后
  ft = "specific-type",   -- 文件类型触发
  cmd = "PluginCommand", -- 命令触发
}

-- 3. 避免同步操作
vim.defer_fn(function()
  -- 异步执行耗时操作
end, 0)
```

### 向后兼容性维护

```lua
-- util/plugin.lua中维护兼容性映射
M.renames = {
  ["old-plugin-name"] = "new-plugin-name",
}

M.deprecated_extras = {
  "lazyvim.plugins.extras.old.extra",
}

-- 提供废弃警告
LazyVim.deprecate("old_function", "new_function", { version = "next-version" })
```

## 总结

LazyVim的开发贡献建立在以下原则之上：

1. **代码质量优先** - 严格的格式化和静态分析要求
2. **测试驱动开发** - 所有功能必须有相应测试
3. **用户体验至上** - 配置必须可覆盖，功能必须直观
4. **性能意识** - 懒加载优化，避免启动性能影响
5. **社区友好** - 清晰的文档和贡献流程

**开发者路径建议**：

1. **熟悉项目** - 深入理解LazyVim架构和设计理念
2. **小处着手** - 从文档改进或小bug修复开始
3. **参与讨论** - 在Issue和Discussion中活跃参与
4. **专业化发展** - 专注某个领域（如Lang extras或UI主题）
5. **长期贡献** - 成为核心贡献者，参与项目治理

通过遵循这些指导原则和最佳实践，开发者可以为LazyVim生态系统做出有价值的贡献，同时提升自己的Neovim和Lua开发技能。
