# LazyVim 用户使用指南

*从零开始使用LazyVim - 完整的安装、配置和使用教程*

---

## 概述

LazyVim 是一个基于 lazy.nvim 的 Neovim 配置框架，为开发者提供开箱即用的IDE体验。它不是最终配置，而是可扩展的配置基础，支持用户根据需求进行个性化定制。

**核心特色**：
- 🚀 **开箱即用** - 预配置常用插件和功能
- ⚡ **极速启动** - 懒加载机制优化性能  
- 🎯 **模块化** - 147个可选功能模块
- 🔧 **易定制** - 用户配置自动覆盖默认设置
- 📱 **智能推荐** - 根据项目类型自动推荐相关插件

## 安装流程

### 系统要求

根据健康检查系统要求（`lua/lazyvim/health.lua`）：

**必需环境**：
- **Neovim >= 0.11.0** （必须，LuaJIT构建）
- **Git >= 2.19.0** （必须，支持部分克隆）

**推荐工具**（可选，提升体验）：
- **ripgrep (rg)** - 快速文本搜索
- **fd/fdfind** - 快速文件查找  
- **lazygit** - Git TUI界面
- **fzf** - 模糊查找
- **curl** - 网络请求
- **Nerd Font** - 图标显示支持
- **C编译器** - TreeSitter语法高亮

### 安装步骤

⚠️ **重要警告**：永远不要直接使用LazyVim框架仓库！必须使用官方Starter模板。

**Step 1: 备份现有配置**
```bash
# 备份现有Neovim配置
mv ~/.config/nvim ~/.config/nvim.bak
mv ~/.local/share/nvim ~/.local/share/nvim.bak
mv ~/.local/state/nvim ~/.local/state/nvim.bak
mv ~/.cache/nvim ~/.cache/nvim.bak
```

**Step 2: 克隆Starter模板**
```bash
# 克隆官方starter模板
git clone https://github.com/LazyVim/starter ~/.config/nvim

# 删除.git目录以便自己管理版本控制
rm -rf ~/.config/nvim/.git
```

**Step 3: 启动和安装**
```bash
# 首次启动Neovim（会自动安装插件）
nvim
```

首次启动时，LazyVim会：
1. 自动安装lazy.nvim插件管理器
2. 安装所有核心插件
3. 配置LSP服务器和语法高亮
4. 设置默认键位映射和选项

### 安装验证

**健康检查**：
```vim
:checkhealth LazyVim
```

**预期输出**：
```
LazyVim: health#check
- Using Neovim >= 0.11.0: ✓

- `rg` is installed: ✓  
- `fd` is installed: ✓
- `lazygit` is installed: ✓ (optional)
- `fzf` is installed: ✓ (optional)
- `curl` is installed: ✓
```

## 配置结构

### Starter模板结构

```
~/.config/nvim/
├── lua/
│   ├── config/
│   │   ├── autocmds.lua    # 自定义自动命令
│   │   ├── keymaps.lua     # 自定义键位映射
│   │   ├── lazy.lua        # lazy.nvim插件管理器配置 
│   │   └── options.lua     # Neovim选项设置
│   └── plugins/
│       └── example.lua     # 自定义插件配置示例
├── .gitignore              # Git忽略文件
├── .neoconf.json          # 项目LSP配置
├── LICENSE                # 开源协议
├── README.md              # 项目说明
├── init.lua               # Neovim入口文件
└── stylua.toml           # Lua代码格式化配置
```

### 配置加载顺序

LazyVim使用分层配置系统：

1. **LazyVim默认配置** - 框架提供的基础配置
2. **Extra插件配置** - 可选功能模块的配置
3. **用户配置覆盖** - `~/.config/nvim/lua/config/` 中的自定义配置

用户配置会自动覆盖框架默认配置，无需手动合并。

### 启用Extra插件

**方法1：LazyExtras交互界面**
```vim
:LazyExtras
```

在界面中：
- 按 `x` 切换插件启用/禁用状态
- 按 `<CR>` 或 `<Space>` 查看插件详情
- 按 `<Tab>` 预览插件规范
- 推荐的插件会有特殊标识

**方法2：配置文件导入**

在 `~/.config/nvim/lua/config/lazy.lua` 中添加：

```lua
require("lazy").setup({
  spec = {
    -- 导入LazyVim核心插件
    { "LazyVim/LazyVim", import = "lazyvim.plugins" },
    
    -- 导入额外插件
    { import = "lazyvim.plugins.extras.lang.typescript" },
    { import = "lazyvim.plugins.extras.lang.python" },
    { import = "lazyvim.plugins.extras.ai.copilot" },
    { import = "lazyvim.plugins.extras.formatting.prettier" },
    
    -- 导入用户插件
    { import = "plugins" },
  },
})
```

### 智能推荐机制

LazyVim会根据项目特征自动推荐相关插件：

**基于文件类型推荐**：
- 打开Python文件 → 推荐Python language extra
- 打开TypeScript文件 → 推荐TypeScript language extra

**基于项目根文件推荐**：
- 发现 `package.json` → 推荐Node.js相关工具
- 发现 `pyproject.toml` → 推荐Python工具链
- 发现 `Cargo.toml` → 推荐Rust工具

**查看推荐状态**：
```vim
:LazyExtras  " 查看所有extras，推荐的会高亮显示
```

## 配置覆盖

### 基本覆盖方式

**选项设置** (`config/options.lua`)：
```lua
-- 覆盖默认选项
vim.opt.conceallevel = 0        -- 显示隐藏字符
vim.opt.showtabline = 2         -- 总是显示标签页
vim.opt.laststatus = 3          -- 全局状态行
```

**键位映射** (`config/keymaps.lua`)：
```lua
-- 添加自定义键位
vim.keymap.set("n", "<leader>xx", "<cmd>TroubleToggle<cr>", { desc = "Toggle Trouble" })

-- 覆盖默认键位
vim.keymap.set("n", "<C-h>", "<C-w>h", { desc = "Go to left window", remap = true })
```

**自动命令** (`config/autocmds.lua`)：
```lua
-- 添加自动命令
local function augroup(name)
  return vim.api.nvim_create_augroup("lazyvim_" .. name, { clear = true })
end

vim.api.nvim_create_autocmd("FileType", {
  group = augroup("markdown_settings"),
  pattern = { "markdown" },
  callback = function()
    vim.opt_local.spell = true
    vim.opt_local.conceallevel = 2
  end,
})
```

### 插件配置覆盖

**方法1：opts覆盖**

在 `plugins/` 目录下创建文件：

```lua
-- plugins/neo-tree.lua
return {
  "nvim-neo-tree/neo-tree.nvim",
  opts = {
    window = {
      width = 30,        -- 覆盖窗口宽度
      position = "right", -- 覆盖位置
    },
    filesystem = {
      filtered_items = {
        hide_dotfiles = false,  -- 显示隐藏文件
      },
    },
  },
}
```

**方法2：config函数完全控制**

```lua
-- plugins/telescope.lua
return {
  "nvim-telescope/telescope.nvim",
  config = function()
    require("telescope").setup({
      defaults = {
        -- 完全自定义配置
        layout_strategy = "vertical",
        layout_config = {
          vertical = { width = 0.9, height = 0.9, preview_height = 0.6 },
        },
      },
    })
  end,
}
```

**方法3：opts函数动态修改**

```lua
-- plugins/nvim-cmp.lua
return {
  "hrsh7th/nvim-cmp",
  opts = function(_, opts)
    local cmp = require("cmp")
    -- 在现有配置基础上添加
    table.insert(opts.sources, { name = "emoji" })
    -- 修改键位映射
    opts.mapping["<Tab>"] = cmp.mapping.select_next_item()
    return opts
  end,
}
```

## 故障排除

### 常见问题

**1. 启动缓慢**
```vim
:Lazy profile  " 查看插件加载时间
```
解决方案：检查是否有插件配置错误或网络问题

**2. LSP不工作**
```vim
:LspInfo      " 查看LSP服务器状态
:Mason        " 管理LSP服务器安装
```
解决方案：确保相关语言的LSP服务器已安装

**3. 格式化不工作**
```vim
:ConformInfo  " 查看格式化器状态
```
解决方案：检查格式化工具是否已安装

**4. 键位冲突**
```vim
:map <leader>  " 查看leader键映射
```
解决方案：在 `config/keymaps.lua` 中重新定义键位

### 健康检查

**全面检查**：
```vim
:checkhealth
```

**LazyVim特定检查**：
```vim
:checkhealth LazyVim
```

**外部工具检查**：

## 高级配置

### 默认选择覆盖

LazyVim在某些功能上提供多个选择，通过全局变量控制：

```lua
-- config/options.lua
vim.g.lazyvim_picker = "telescope"    -- 强制使用telescope而非fzf
vim.g.lazyvim_cmp = "nvim-cmp"        -- 强制使用nvim-cmp而非blink.cmp
vim.g.lazyvim_tree = "neo-tree"       -- 文件浏览器选择
```

### 禁用默认插件

```lua
-- plugins/disabled.lua
return {
  -- 禁用默认插件
  { "folke/flash.nvim", enabled = false },
  { "echasnovski/mini.pairs", enabled = false },
}
```

### 条件加载插件

```lua
-- plugins/conditional.lua
return {
  -- 只在特定条件下加载
  {
    "some-plugin/repo",
    enabled = function()
      return vim.fn.executable("some-tool") == 1
    end,
  },
  
  -- 只在特定文件类型下加载
  {
    "filetype-plugin/repo", 
    ft = { "python", "lua" },
  },
}
```

### 自定义Extras

创建自己的extra插件模块：

```lua
-- plugins/extras/my-extra.lua
return {
  {
    "my-plugin/repo",
    config = function()
      -- 配置逻辑
    end,
  },
}
```

## 最佳实践

### 配置管理

1. **版本控制** - 将 `~/.config/nvim` 加入Git管理
2. **分离关注点** - 不同功能的配置放在不同文件
3. **渐进配置** - 从默认配置开始，逐步添加自定义
4. **定期更新** - 使用 `:Lazy update` 更新插件

### 性能优化

1. **按需启用Extra** - 只启用真正需要的功能
2. **懒加载优先** - 配置插件时使用适当的加载时机
3. **定期清理** - 删除不用的插件和配置
4. **性能监控** - 使用 `:Lazy profile` 监控加载时间

### 学习路径

1. **熟悉基础** - 先掌握基本的编辑操作和导航

3. **自定义配置** - 根据需求调整键位和选项
