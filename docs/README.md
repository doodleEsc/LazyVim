## 🛠️ 专项资源

### [技术决策记录 (ADR)](./zh/技术决策记录/)
> *重要技术决策的背景和理由*

记录 LazyVim 发展过程中的关键技术决策，帮助维护者理解设计背景和权衡考虑。

**使用指南**：
- 📝 每个重要决策创建一个ADR文件
- 🔢 使用序号命名：`001-决策标题.md`
- 📅 包含决策时间、背景、方案比较和结果
- 🔄 定期回顾和更新过时的决策

### [故障排除指南](./zh/故障排除指南.md)
> *常见问题和解决方案集合*

基于健康检查系统和实际使用经验总结的问题诊断指南。

**主要内容**：
- ⚠️ 安装和环境问题
- 🔧 配置和插件冲突
- 🐛 常见错误和解决方法
- 📊 性能诊断和优化建议

### 文档改进

欢迎对维护者文档提出改进建议：

1. **内容更新** - 发现过时或错误信息
2. **结构优化** - 改善文档组织和导航
3. **案例补充** - 添加实际使用案例和示例
4. **翻译工作** - 多语言文档支持

### 反馈渠道

- **GitHub Issues** - 报告文档问题或建议改进
- **GitHub Discussions** - 讨论文档架构和内容策略
- **Pull Requests** - 直接贡献文档改进

### 文档写作规范

- **中文优先** - 维护者文档以中文为主要语言
- **结构清晰** - 使用层次化标题和导航
- **示例丰富** - 提供具体的代码示例和操作步骤
- **及时更新** - 随着项目发展持续更新内容
